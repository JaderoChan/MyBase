# Markdown 文档空格标准

Markdown 文档编写中需要通过一个空格对以下内容进行分隔。（以下内容中出现的 **正文** 均指代中文正文（因为在英文正文中，空格本身就常用于各种内容之间的分隔），且不包含标点符号（无论半角还是全角符号））

对于符号与符号之间的空格，可以根据上下文决定是否保留空格。例如以下情况是被允许且推荐的：

```markdown
使用 `stdint.h`（C）/ `cstdint`（C++）头文件。

（斜杠与反引号 “）/ `cstd” 之间保留一个空格。因为保留空格后，斜杠右侧的空白区域能与左侧右括号的空白区域保持“平衡”）
```

## 必须

- 西语字母、单词与正文

    **例外情况**

    - 当某个西语字母或单词与某个对象是强相关的，互相绑定时

        - `此文档的作者是𬱖珞JaderoChan` 适用此种情况。（𬱖珞与 JaderoChan 是强相关，互相绑定的）

- 代码片段与正文
- 公式片段与正文

## 建议

- 文档高亮或强调部分与正文

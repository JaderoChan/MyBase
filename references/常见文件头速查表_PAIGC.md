## 基础说明

| 术语 | 含义 |
| --- | --- |
| 文件头 / 魔数（Magic Number） | 文件起始若干字节，用于快速识别文件类型 |
| 偏移（Offset） | 签名不一定在第 0 字节，可能在固定偏移位置 |
| `??` | 通配任意字节（此文档中的约定写法） |

- 表格中的十六进制字节默认使用大写并以空格分隔，如：`89 50 4E 47 0D 0A 1A 0A`。
- 若某组 Hex 可直接组成可读字符，会列在“可读字符（ASCII）”列。

| Shebang 脚本 | `23 21` | `0` | `#!` | 常见于 shell/python 脚本 |

## 实用示例

### 用 `file` 快速识别

```bash
file sample.bin
```

### 查看前 32 字节（十六进制）

```bash
xxd -g 1 -l 32 sample.bin
```

### 仅输出前 16 字节为连续十六进制

```bash
xxd -p -l 16 sample.bin
```

### 检查 TAR 的 `ustar` 偏移签名

```bash
xxd -g 1 -s 257 -l 5 archive.tar
```

## 注意事项

- 文件头只能做快速识别，不能替代完整格式校验。
- 容器格式（如 `RIFF`、`ISO BMFF`）需要结合后续字段判断子类型。
- 同一签名可能对应多个类型（如 `CA FE BA BE` 可能是 Java Class，也可能是 Fat Mach-O）。
- 某些文件可能被追加数据或伪造头部，安全场景需做深度解析。

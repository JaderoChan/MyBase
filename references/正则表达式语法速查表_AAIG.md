## 基础概念

| 术语 | 含义 | 示例 |
| --- | --- | --- |
| 字面量匹配 | 按字符原样匹配 | `cat` 匹配 `cat` |
| 模式（Pattern） | 正则表达式本体 | `^[a-z]+$` |
| 输入串（Subject） | 被匹配文本 | `hello` |
| 匹配（Match） | 模式命中的片段 | `\d+` 命中 `123` |
| 捕获组（Capture） | 用 `()` 保存子匹配 | `(ab)+` |

## 元字符与转义

下列字符在正则中有特殊意义：

```text
. ^ $ * + ? ( ) [ ] { } | \
```

若要匹配它们本身，通常需要加反斜杠转义：

| 目标字符 | 写法 |
| --- | --- |
| `.` | `\.` |
| `*` | `\*` |
| `?` | `\?` |
| `(` | `\(` |
| `[` | `\[` |
| `\` | `\\` |

## 字符类与预定义字符类

### 字符类

| 写法 | 含义 | 示例 |
| --- | --- | --- |
| `[abc]` | 匹配 `a`/`b`/`c` 任一字符 | `[abc]+` |
| `[^abc]` | 匹配不在集合内的任一字符 | `[^0-9]` |
| `[a-z]` | 范围（小写字母） | `[a-zA-Z_]` |
| `[0-9]` | 数字范围 | `[0-9]{4}` |

### 预定义字符类

| 写法 | 含义 | 近似等价 |
| --- | --- | --- |
| `\d` | 数字字符 | `[0-9]` |
| `\D` | 非数字字符 | `[^0-9]` |
| `\w` | 单词字符（字母数字下划线） | `[A-Za-z0-9_]`（多数引擎） |
| `\W` | 非单词字符 | `[^A-Za-z0-9_]`（多数引擎） |
| `\s` | 空白字符 | `[ \t\r\n\f\v]`（常见） |
| `\S` | 非空白字符 | `[^\s]` |

### 点号

| 写法 | 含义 | 备注 |
| --- | --- | --- |
| `.` | 匹配除换行外的任意字符 | 开启 dotall（`s`）后通常可匹配换行 |

## 量词（重复次数）

| 写法 | 含义 | 说明 |
| --- | --- | --- |
| `*` | `0` 次或多次 | 贪婪 |
| `+` | `1` 次或多次 | 贪婪 |
| `?` | `0` 次或 `1` 次 | 贪婪 |
| `{n}` | 恰好 `n` 次 | - |
| `{n,}` | 至少 `n` 次 | - |
| `{n,m}` | `n` 到 `m` 次 | - |
| `*?` `+?` `??` `{n,m}?` | 懒惰量词 | 尽可能少匹配 |
| `*+` `++` `?+` `{n,m}+` | 占有量词 | 部分引擎支持（如 PCRE、Java） |

## 分组、引用与命名组

| 写法 | 含义 | 备注 |
| --- | --- | --- |
| `(abc)` | 捕获组 | 可通过组号引用 |
| `(?:abc)` | 非捕获组 | 只分组，不捕获 |
| `(?<name>abc)` | 命名捕获组 | 常见于 PCRE、.NET、JavaScript |
| `(?P<name>abc)` | 命名捕获组 | Python 常用写法 |
| `\1` `\2` | 反向引用（组号） | 匹配与前面组相同文本 |
| `\k<name>` | 反向引用（组名） | 语法依引擎而异 |

## 锚点与边界

| 写法 | 含义 | 备注 |
| --- | --- | --- |
| `^` | 行/字符串开头 | 多行模式下表示每一行开头 |
| `$` | 行/字符串结尾 | 多行模式下表示每一行结尾 |
| `\A` | 字符串绝对开头 | 部分引擎支持 |
| `\Z` | 字符串绝对结尾（末尾换行前） | 部分引擎支持 |
| `\z` | 字符串绝对结尾 | 部分引擎支持 |
| `\b` | 单词边界 | `\w` 与 `\W` 交界处 |
| `\B` | 非单词边界 | 不是 `\b` 的位置 |

## 断言（零宽）

| 写法 | 含义 | 例子 |
| --- | --- | --- |
| `(?=...)` | 正向先行断言 | `\w+(?=:)` |
| `(?!...)` | 负向先行断言 | `foo(?!bar)` |
| `(?<=...)` | 正向后行断言 | `(?<=\$)\d+` |
| `(?<!...)` | 负向后行断言 | `(?<!-)\d+` |

- 断言只判断位置，不消耗字符。
- 某些引擎对后行断言长度有限制（例如旧版本 JavaScript 不支持后行断言）。

## 修饰符（Flags）

| Flag | 含义 | 常见影响 |
| --- | --- | --- |
| `i` | 忽略大小写 | `a` 可匹配 `A` |
| `m` | 多行模式 | `^`、`$` 按行生效 |
| `s` | dotall 模式 | `.` 可匹配换行 |
| `g` | 全局匹配 | 查找所有命中（常见于 JavaScript） |
| `u` | Unicode 模式 | 影响字符边界与转义解析 |
| `x` | 扩展模式 | 忽略模式中的空白和注释（部分引擎） |

## 替换时常用占位符

不同语言写法略有差异，以下是常见风格。

| 写法 | 含义 | 常见场景 |
| --- | --- | --- |
| `$0` 或 `$&` | 整个匹配 | JavaScript 常见 |
| `$1` `$2` | 第 `1`/`2` 个捕获组 | 多数引擎支持 |
| `${name}` | 命名组引用 | JavaScript、部分引擎 |
| `\1` | 第 `1` 个捕获组 | Python、部分工具 |
| `\g<name>` | 命名组引用 | Python 常用 |

## 常见引擎差异

| 特性 | PCRE | JavaScript（ES2018+） | Python `re` |
| --- | --- | --- | --- |
| 命名组 | `(?<name>...)` | `(?<name>...)` | `(?P<name>...)` |
| 后行断言 | 支持 | 支持（现代环境） | 支持 |
| 占有量词 | 支持 | 不支持 | 不支持 |
| 原子组 `(?>...)` | 支持 | 不支持 | 不支持 |
| 扩展模式 `x` | 支持 | 不支持（可用其他方式） | 支持（`re.X`） |

## 示例

| 目标 | 模式 | 备注 |
| --- | --- | --- |
| 整数 | `^[+-]?\d+$` | 可带正负号 |
| 小数 | `^[+-]?(?:\d+\.\d+\|\d+\|\.\d+)$` | 支持 `1`、`1.2`、`.5` |
| 16 进制颜色 | `^#(?:[0-9A-Fa-f]{3}\|[0-9A-Fa-f]{6})$` | `#fff` 或 `#ffffff` |
| 用户名（字母开头，3-16 位） | `^[A-Za-z][A-Za-z0-9_]{2,15}$` | 仅示例规则 |
| 空白行 | `^\s*$` | 匹配全空白或空行 |
| 去首尾空白（查找） | `^\s+\|\s+$` | 常用于替换为空 |
| 重复单词 | `\b(\w+)\b\s+\b\1\b` | 如 `go go` |
| IPv4（宽松） | `^(?:\d{1,3}\.){3}\d{1,3}$` | 不保证每段 `<=255` |
| 邮箱（宽松） | `^[^\s@]+@[^\s@]+\.[^\s@]+$` | 工程上常用宽松校验 |

## 调试与性能建议

- 先写最小可用模式，再逐步加约束，避免一次写得过度复杂。
- 能加锚点时优先加锚点（`^...$`），减少回溯范围。
- 警惕灾难性回溯：避免多层可变重复相互嵌套，如 `(a+)+`。
- 对大型文本优先使用非捕获组 `(?:...)`，减少无意义捕获。
- 若引擎支持，可使用原子组或占有量词降低回溯开销。
- 跨语言迁移时，先确认引擎差异（命名组语法、后行断言、Unicode 行为）。
